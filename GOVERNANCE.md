# Governance

MAL UI is maintained by MAL Devs with help from the open-source community.

## Project Goals

- Provide a reliable React component library built on Mantine.
- Keep MAL brand theme, tokens, and exports consistent across projects.
- Make contributions approachable through clear docs, templates, and review
  expectations.
- Preserve compatibility for consumers whenever possible.

## Roles

### Contributors

Anyone who opens issues, improves documentation, reviews pull requests, tests
changes, or contributes code.

### Maintainers

People with repository write access. Maintainers triage issues, review pull
requests, manage releases, and make final decisions when consensus is not clear.

## Decision Making

Most decisions should happen in public issues or pull requests. Maintainers
prefer consensus, but may make final calls to keep the project moving.

Changes with broad impact should include:

- A clear problem statement.
- Compatibility notes.
- Migration guidance, if needed.
- Tests or examples that show the new behavior.

## Becoming a Maintainer

Regular contributors may be invited to become maintainers when they consistently
show good judgment, respectful communication, and care for project quality.

## Conflict Resolution

If a discussion becomes unproductive, maintainers may pause it, summarize the
decision points, and ask participants to return with concrete proposals.
Behavior issues are handled under the Code of Conduct.
