# Roadmap

This roadmap is a guide, not a promise. Priorities can change based on user
needs, maintainer capacity, and community contributions.

## Near Term

- Improve public documentation and examples for every subpath.
- Add more focused tests around package exports and theme behavior.
- Expand MAL-branded aliases and helpers where they provide real value.
- Keep Mantine packages upgraded in lockstep.

## Component Library Quality

- Strengthen accessibility guidance and checks.
- Add visual examples for common layout and form patterns.
- Document migration notes for breaking changes.
- Keep bundle outputs small and tree-shakeable.

## Community

- Label beginner-friendly issues.
- Keep issue templates current.
- Improve release notes and changelog quality.
- Encourage small, focused pull requests from contributors.

## Not Planned Right Now

- Replacing Mantine as the underlying component system.
- Adding framework-specific runtime code to the core package.
- Shipping private MAL application logic in this public package.
